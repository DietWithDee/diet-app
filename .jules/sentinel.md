## 2024-05-28 - [XSS via dangerouslySetInnerHTML]
**Vulnerability:** The application was using React's `dangerouslySetInnerHTML` to render user-generated or fetched HTML directly into the DOM (e.g. Email Template Builder, Articles Manager, Blog). This opened up the possibility of Cross-Site Scripting (XSS) if malicious scripts were included in the HTML strings.
**Learning:** `dangerouslySetInnerHTML` must always be treated with caution, especially when handling data originating outside a controlled code path. In some files, string truncation on HTML content (`substring(0, 150)`) was also being performed, which can easily create malformed, unclosed tags that break page layout when rendered via `dangerouslySetInnerHTML`.
**Prevention:** Always use an HTML sanitization library (like `dompurify`) when injecting dynamic HTML using `dangerouslySetInnerHTML`. Ensure that `DOMPurify.sanitize()` is called *after* any string manipulation (like truncation) to prevent both XSS attacks and malformed markup issues.
